# KnowFlow 技术架构设计

## 1. 技术栈推荐

### 1.1 前端技术栈
- **框架**: React + TypeScript
- **状态管理**: Redux Toolkit
- **UI组件库**: Ant Design
- **文件处理**: electron (桌面端) / File API (浏览器端)
- **构建工具**: Vite
- **样式**: CSS-in-JS (styled-components)

### 1.2 后端技术栈
- **运行环境**: Node.js
- **API框架**: Express
- **文件系统操作**: fs-extra
- **网络请求**: axios
- **安全**: helmet, cors

### 1.3 数据库
- **主存储**: SQLite (轻量级，适合本地应用)
- **缓存**: Redis (可选，用于提高检索性能)

## 2. 核心模块划分

### 2.1 前端模块
- **导航模块**: 左侧key分类导航
- **检索模块**: 正则表达式检索，支持多key组合
- **导入模块**: 本地文件/文件夹导入
- **详情模块**: 文件key-value展示
- **插件管理模块**: 插件的加载与配置

### 2.2 后端模块
- **文件处理模块**: 文件导入、读取、元数据提取
- **Key-Value管理模块**: 内置key管理、自定义key管理
- **检索模块**: 正则匹配、排序
- **插件系统模块**: 插件加载、生命周期管理
- **AI集成模块**: 调用大模型API

### 2.3 核心模块
- **数据存储模块**: 数据库操作、缓存管理
- **文件系统模块**: 文件索引、路径管理
- **插件接口模块**: 插件标准化接口定义

## 3. 数据存储方案

### 3.1 数据库表结构

#### 3.1.1 `knowledge_items` 表
| 字段名 | 数据类型 | 描述 |
|-------|---------|------|
| `id` | INTEGER | 知识项ID (主键) |
| `file_path` | TEXT | 文件路径 |
| `file_type` | TEXT | 文件类型 |
| `created_at` | TIMESTAMP | 创建时间 |
| `click_count` | INTEGER | 点击次数 |
| `star_rating` | INTEGER | 星级 (1-5) |
| `metadata` | JSON | 其他元数据 |

#### 3.1.2 `key_categories` 表
| 字段名 | 数据类型 | 描述 |
|-------|---------|------|
| `id` | INTEGER | 分类ID (主键) |
| `name` | TEXT | 分类名称 |
| `parent_id` | INTEGER | 父分类ID |

#### 3.1.3 `key_definitions` 表
| 字段名 | 数据类型 | 描述 |
|-------|---------|------|
| `id` | INTEGER | Key定义ID (主键) |
| `name` | TEXT | Key名称 |
| `category_id` | INTEGER | 所属分类ID |
| `is_builtin` | BOOLEAN | 是否内置 |
| `script` | TEXT | 小脚本代码 |

#### 3.1.4 `key_values` 表
| 字段名 | 数据类型 | 描述 |
|-------|---------|------|
| `id` | INTEGER | 记录ID (主键) |
| `item_id` | INTEGER | 知识项ID |
| `key_id` | INTEGER | Key定义ID |
| `value` | TEXT | Value值 |

### 3.2 索引策略
- 为 `file_path`、`file_type`、`click_count`、`star_rating` 等常用字段创建索引
- 为 `key_values` 表的 `item_id` 和 `key_id` 字段创建复合索引，提高检索性能

### 3.3 插件存储
- 插件存储在 `plugins` 目录下，每个插件为独立子目录
- 插件配置存储在 `plugin_config.json` 文件中

## 5. 关键流程

### 5.1 文件导入流程
1. 用户选择文件/文件夹
2. 前端读取文件元数据
3. 后端处理文件，提取内置key
4. 存储到数据库
5. 前端更新界面

### 5.2 检索流程
1. 用户输入检索条件
2. 前端解析检索表达式
3. 后端执行数据库查询
4. 按排序条件排序
5. 前端展示结果

### 5.3 插件调用流程
1. 前端触发插件操作
2. 后端加载对应插件
3. 执行插件逻辑
4. 返回结果
5. 更新知识库

## 6. 扩展性考虑

- **模块化设计**: 各模块独立，便于扩展
- **插件系统**: 支持动态加载新功能
- **API设计**: 前后端通过RESTful API通信，便于后续扩展
- **数据模型**: 支持自定义key，满足不同用户需求

## 7. 性能优化策略

- **缓存机制**: 常用数据缓存
- **索引优化**: 合理创建数据库索引
- **异步处理**: 文件导入等耗时操作异步执行
- **批量操作**: 批量处理文件导入，减少数据库操作次数
