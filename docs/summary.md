# KnowFlow 项目总结

## 项目概述

**KnowFlow** 是一个知识管理系统，旨在帮助用户高效管理、检索和组织各类知识资源。系统采用前后端分离架构，支持文件导入、动态属性管理、插件扩展和 AI 语义检索等功能。

## 核心功能模块

### 1. 知识项管理

- **文件上传**: 支持拖拽上传、自动提取文件元数据
- **CRUD 操作**: 完整的创建、读取、更新、删除功能
- **动态属性**: 基于 Key-Value 系统的灵活属性管理
- **文件预览**: 支持图片、视频等媒体文件预览

### 2. Key-Value 系统

- **Key 定义**: 支持多种数据类型（string, number, boolean, array, object）
- **分类管理**: 层级分类结构，支持父子关系
- **内置 Key**: 预定义的基础属性、时间属性等
- **自定义 Key**: 用户可创建自定义属性

### 3. 搜索与检索

- **关键词搜索**: 支持按名称、路径等字段搜索
- **排序功能**: 按点击次数、星级、时间等排序
- **AI 语义检索**: 集成豆包 AI 进行语义理解检索

### 4. 插件系统

- **动态加载**: 运行时加载和卸载插件
- **标准化接口**: 统一的插件接口定义
- **插件类型**: 知识源导入、检索、知识处理、可视化、自动化
- **内置插件**: 星级评分插件

### 5. AI 集成

- **语义检索**: 自然语言查询知识库
- **自动标签**: AI 自动生成标签（预留功能）

## 数据模型

### 知识项 (items)

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | ObjectId | 唯一标识 |
| `name` | string | 知识项名称 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |
| `file_path` | string | 文件路径 |
| `file_type` | string | 文件类型 |
| 动态属性 | various | Key-Value 系统 |

### 分类 (categories)

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | ObjectId | 唯一标识 |
| `name` | string | 分类名称 |
| `title` | string | 显示标题 |
| `parent_name` | string | 父分类名称 |
| `is_builtin` | boolean | 是否内置 |

### Key 定义 (keys)

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | ObjectId | 唯一标识 |
| `name` | string | Key 名称 |
| `title` | string | 显示标题 |
| `value_type` | string | 数据类型 |
| `category_name` | string | 所属分类 |
| `is_required` | boolean | 是否必填 |
| `is_visible` | boolean | 是否可见 |
| `plugin_name` | string | 所属插件 |

## API 接口概览

### 基础信息

- **Base URL**: `http://localhost:3000/api/v1`
- **CORS**: 已启用跨域支持

### 主要接口

| 模块 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 健康检查 | `/health` | GET | 服务状态检查 |
| 知识项 | `/item` | GET/POST | 列表/创建 |
| 知识项 | `/item/{id}` | GET/PUT/DELETE | 详情/更新/删除 |
| 文件上传 | `/upload` | POST | 文件上传 |
| 分类 | `/categories` | GET/POST | 列表/创建 |
| 分类 | `/categories/{id}` | GET/PUT/DELETE | 详情/更新/删除 |
| Key | `/keys` | GET/POST | 列表/创建 |
| Key | `/keys/{name}` | GET/PUT/DELETE | 详情/更新/删除 |
| 插件 | `/plugins/manifests` | GET | 插件清单 |
| 插件 | `/plugins/{name}/frontend` | GET | 插件前端代码 |

## 部署方式

### 开发环境

```bash
# 后端
cd backend
pip install -r requirements.txt
python main.py

# 前端
cd frontend
npm install
npm run dev
```

### Docker 部署

```bash
cd backend
docker-compose up -d
```

### Electron 打包

```bash
cd frontend
npm run electron:build
```
